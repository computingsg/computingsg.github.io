01 · The build

What you'll actually build.

Not a slide deck. A working bot. By the end of the workshop you'll be messaging your own AI on Telegram, watching it organize sources into a structured wiki on your own machine.

  1. Install nanobot & Telegram channel

    One command via pipx. Onboard a workspace, register a Telegram bot with @BotFather, scope it to your user ID only.

  2. Configure the LLM (cloud or local)

    Path A: OpenRouter for fast cloud replies (free tier available). Path B: Local Ollama for privacy and offline operation. Both supported as first-class paths.

  3. Write the schema that defines the bot's job

    A tight AGENTS.md turns a generic chatbot into a disciplined wiki maintainer. MANDATORY rules. Verbatim quotes. Scoped refusals.

  4. Ingest sources, query the wiki, watch compounding

    Paste articles or drag-drop PDFs into Telegram. The bot extracts, files, and cites. Every source teaches the brain multiple things.

02 · Prerequisites

Before you show up.

Bring a machine, install nothing (unless you want to save time — the worksheet walks through installs from scratch).

  • A machine — Mac, Linux, Windows (WSL), or a Raspberry Pi 4/5 4 GB RAM minimum, 8+ GB recommended. Workshop has been tested on Pi 5 16 GB.
  • Python ≥ 3.11 Check with python3 --version. Most modern systems have this.
  • A Telegram account On your phone. You'll create a bot with @BotFather during the workshop.
  • An OpenRouter account (optional) Free tier covers the workshop. Sign up at openrouter.ai. Skip if you're going local-only.
  • A topic in mind Something narrow you care about — your craft, a learning goal, your job role. The worksheet's default running example is AI fluency.
